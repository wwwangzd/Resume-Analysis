# 项目记忆

更新时间：2026-05-08

## 当前基线

- 主链路固定为：`utils/document.py` 文档文本提取 -> `src/llm_extract.py` 结构化抽取 -> `src/script.py` API 编排。
- 文档提取策略固定为“PDF 原生文本提取优先，OCR 兜底回退”。
- 输出固定为六个模块：基础信息、实习/工作经历、项目经历、获奖信息、自我评价、其他。
- 默认 OCR 参数为 `pdf_render_scale = 1.0`、`use_angle_cls = false`、`use_gpu = false`。
- LLM 输入预处理默认开启噪声过滤、连续重复行去重、空白规范化和保守裁剪。

## 已完成优化

1. 已补齐请求读取、文档提取、LLM 预处理、LLM 调用、缓存命中、响应格式化和总耗时等阶段日志。
2. 已完成原生文本优先、OCR 回退的文档提取策略。
3. 已完成 OCR 成本收敛，当前默认走低成本参数。
4. 已完成 LLM 文本清洗、去重、压缩和裁剪。
5. 已完成 `ChatTongyi` 实例、`ChatPromptTemplate` 和 chain 的进程内复用。
6. 已为 LLM 实例缓存、提示模板缓存和 chain 缓存补齐线程安全保护。
7. 已将同步 LLM 调用放入线程池，避免单次调用长时间阻塞事件循环。
8. 已将 `uvicorn` 默认 worker 数配置为 2，并统一由 `config/settings.json` 管理本地与容器启动配置。

## 后续优化顺序

1. 增加 LLM 并发上限、排队机制和超时保护。
2. 评估多 worker 模式下 OCR 实例、LLM 缓存与内存占用情况。
3. 将 OCR 回退路径从整份 PDF 全量渲染优化为逐页渲染、逐页处理、逐页释放。
4. 增加文件大小、页数、处理时长等请求级保护阈值。
5. 补齐更系统的 benchmark / 压测结果，覆盖冷启动、预热后和 OCR 回退场景。

## 验证样例

- 默认实际测试样例为 `test/刘坤钰.pdf`。
- 优先通过真实 `/analysis` 解析链路验证改动是否生效。
